# Change Management

Control changes to production systems to maintain security and stability.

## Objective

Ensure all changes are reviewed, tested, approved, and documented with rollback capability.

## Implementation

- All changes through version control
- Peer review required
- Test in non-production first
- Change tickets document what/why/who
- Rollback plan for risky changes

## Evidence

- Pull request history
- Change tickets
- Deployment logs
